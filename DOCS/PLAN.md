# План разработки Assembling Manager

## Цель

C#-плагин для Autodesk Revit 2021, который для каждой готовой сборки (`AssemblyInstance`) в модели создаёт виды и применяет фильтр, оставляя на виде только элементы данной сборки.

---

## Общие решения

- **Версия Revit**: 2021
- **Целевой фреймворк**: .NET Framework 4.8 (`net48`)
- **Платформа**: x64
- **UI**: WPF-окно с галочками План / Разрез / 3D
- **Параметр для фильтра**: `AssemblyParameter` (тип Text)
- **Категории параметра**: динамически — по категориям элементов внутри каждой сборки
- **Вложенные семейства**: рекурсивный поиск общих вложенных семейств через `FamilyInstance.GetSubComponentIds()`
- **Старые виды**: удаляются по имени перед созданием новых
- **Разрезы**: два перпендикулярных разреза (вдоль X и вдоль Y)
- **Фильтр**: отдельный `ParameterFilterElement` для каждой сборки, правило «параметр не равен имени сборки», скрыть прошедшие фильтр элементы

---

## Этап 1. Подготовка проекта к Revit 2021

- [ ] В `Directory.Build.props` по умолчанию `RevitVersion=2021`, `TargetFramework=net48`, `PlatformTarget=x64`.
- [ ] Проверить путь `C:\Program Files\Autodesk\Revit 2021\` для `RevitAPI.dll` / `RevitAPIUI.dll`.
- [ ] Убедиться, что `AssemblingManager.Revit` ссылается на `AssemblingManager.Core`.
- [ ] Обновить `.addin` манифест: тип `Application`, уникальный `AddInId`, класс `AssemblingManager.Revit.App : IExternalApplication`.
- [ ] Создать `App.cs` (`IExternalApplication`) с вкладкой ленты, панелью и кнопкой.
- [ ] Создать `CreateAssemblyViewsCommand : IExternalCommand` с атрибутами `TransactionMode.Manual` и `RegenerationOption.Manual`.

## Этап 2. UI — WPF-окно выбора видов

- [ ] Добавить WPF-окно в проект `AssemblingManager.Revit`.
- [ ] Подключить сборки `PresentationCore`, `PresentationFramework`, `WindowsBase`.
- [ ] Создать `MainWindow.xaml` с тремя `CheckBox` (План, Разрез, 3D) и кнопками «Сформировать виды» / «Отмена».
- [ ] Создать `ViewCreationOptions` — объект результата окна.
- [ ] Из команды открывать окно через `ShowDialog()` и передавать выбор в дальнейшую логику.

## Этап 3. Сервисы в `AssemblingManager.Core`

- [ ] `AssemblyService` — сбор элементов сборки, рекурсивный поиск вложенных shared семейств, объединённый BoundingBox, ближайший уровень.
- [ ] `ParameterService` — проверка/создание параметра проекта `AssemblyParameter`, запись значения в элементы, получение `ElementId` параметра для фильтра.
- [ ] `ViewService` — удаление старых видов по имени, создание плана, двух разрезов, 3D вида, обрезка видов.
- [ ] `FilterService` — создание `ParameterFilterElement` для сборки, применение фильтра к виду со скрытием элементов.
- [ ] `OrchestratorService` — фасад, объединяющий всю логику для всех сборок.

## Этап 4. Создание параметра проекта

- [ ] Реализовать создание `AssemblyParameter` через временный shared parameter file (или встроенный ресурс).
- [ ] Тип параметра: `Text`.
- [ ] Привязка: `InstanceBinding` к категориям элементов, найденным в сборках.
- [ ] Проверка существования параметра перед созданием.

## Этап 5. Обработка каждой сборки

- [ ] Собрать все `AssemblyInstance` из документа.
- [ ] Для каждой сборки:
  - [ ] Получить `GetMemberIds()`.
  - [ ] Рекурсивно добавить вложенные общие семейства через `GetSubComponentIds()`.
  - [ ] Записать в параметр `AssemblyParameter` значение `assembly.Name` для всех собранных элементов.

## Этап 6. Создание видов

- [ ] Перед созданием удалить виды с именами:
  - `{ИмяСборки}_План`
  - `{ИмяСборки}_Разрез_1`
  - `{ИмяСборки}_Разрез_2`
  - `{ИмяСборки}_3D`
- [ ] Создать план этажа через `ViewPlan.Create` на ближайшем уровне, обрезать по BoundingBox.
- [ ] Создать два перпендикулярных разреза через `ViewSection.CreateSection` с корректно настроенным `BoundingBoxXYZ`.
- [ ] Создать 3D вид через `View3D.CreateIsometric` и установить `SectionBox` по BoundingBox.

## Этап 7. Создание и применение фильтров

- [ ] Для каждой сборки создать `ParameterFilterElement`:
  - [ ] Имя фильтра: `{ИмяСборки}_Фильтр`.
  - [ ] Категории: категории элементов сборки.
  - [ ] Правило: `AssemblyParameter` не равен `assembly.Name`.
- [ ] Применить фильтр к каждому созданному виду и скрыть прошедшие фильтр элементы (`SetFilterVisibility(filterId, false)`).

## Этап 8. Обработка ошибок

- [ ] Использовать `TransactionGroup` для всей операции.
- [ ] Внутри группы — отдельные `Transaction` для параметра, записи значений, видов, фильтров.
- [ ] При ошибке выполнять `Rollback()`.
- [ ] Подключить `IFailuresPreprocessor` для подавления некритичных предупреждений Revit.
- [ ] Вернуть `Result.Failed` с понятным сообщением.

## Этап 9. Тестирование

- [ ] Собрать решение: `dotnet build build/AssemblingManager.sln /p:RevitVersion=2021`.
- [ ] Скопировать `.dll` и `.addin` в `%APPDATA%\Autodesk\Revit\Addins\2021\`.
- [ ] Запустить Revit 2021 с тестовым проектом, содержащим несколько сборок.
- [ ] Проверить:
  - [ ] Появилась ли вкладка и кнопка.
  - [ ] Открывается ли окно с галочками.
  - [ ] Создаются ли виды с правильными именами.
  - [ ] Корректно ли обрезаны виды.
  - [ ] Скрыты ли лишние элементы фильтром.
  - [ ] Обработаны ли вложенные общие семейства.
  - [ ] Корректно ли работает повторный запуск.

---

## Примечания

- Параметр `AssemblyParameter` создаётся как общий параметр, привязанный к проекту (Revit API не позволяет создать чистый project parameter без shared parameter file).
- Для обрезки плана используется `View.CropBox` (transform игнорируется, учитываются Min/Max).
- Для разрезов и 3D-вида важно правильно задать `BoundingBoxXYZ.Transform`.
- Плагин работает только с уже созданными сборками (`AssemblyInstance`), не создаёт их сам.
