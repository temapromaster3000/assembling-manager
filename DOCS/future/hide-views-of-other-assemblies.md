# Скрытие чужих разрезов/видов на созданных видах

## Проблема

При создании видов для одной сборки на плане и разрезах видны разрезы (и возможно другие виды), созданные для других сборок.

Причина: текущий фильтр `AssemblyParameter` привязан только к категориям элементов сборки (стены, балки и т.п.). Сами виды (`ViewSection`, `ViewPlan`, `View3D`) не получают этот параметр, поэтому фильтр не влияет на них.

## Почему нельзя просто добавить Views к общему параметру

Если привязать `AssemblyParameter` к категории **Views**, фильтр `AssemblyParameter != ИмяСборки` скроет все виды с пустым значением параметра — включая пользовательские виды, созданные вручную. Это неприемлемо.

## Рекомендуемое решение

Создать отдельный параметр `AssemblyView`, привязанный только к категории **Views**, и заполнять его только для видов, созданных плагином.

### 1. Отдельный параметр для видов

- Создать в `ParameterService.cs` метод:
  ```csharp
  public ElementId GetOrCreateViewParameter(Document doc, Application app)
  ```
- Параметр `AssemblyView` типа Text привязать к категории `OST_Views`.
- Это не влияет на пользовательские виды — у них параметр остаётся пустым.

### 2. Фильтр для видов

- В `FilterService.cs` добавить метод:
  ```csharp
  public ParameterFilterElement CreateViewFilter(Document doc, ElementId viewParameterId, string assemblyName)
  ```
- Правило фильтра:
  - `AssemblyView` не пустой
  - `AssemblyView` не равен имени текущей сборки
- Таким образом скрываются только виды других сборок, созданные плагином.

### 3. Установка параметра и применение фильтра

- В `OrchestratorService.cs`:
  - получить/создать `viewParameterId` один раз перед циклом;
  - после создания каждого вида установить ему `AssemblyView = assembly.Name`;
  - создать фильтр для видов и применить его к каждому созданному виду.

### 4. Очистка при удалении старых видов

- В `ViewService.DeleteExistingViews` при удалении старых видов также удалять связанные с ними фильтры для видов, чтобы не копился мусор.

## Файлы, которые затронет изменение

- `src/AssemblingManager.Revit/Services/ParameterService.cs`
- `src/AssemblingManager.Revit/Services/FilterService.cs`
- `src/AssemblingManager.Revit/Services/OrchestratorService.cs`
- `src/AssemblingManager.Revit/Services/ViewService.cs`

## Контрольный список ручной проверки

1. Создать виды для нескольких сборок.
2. Открыть план одной из сборок.
3. Убедиться, что на плане видны:
   - элементы текущей сборки;
   - вид (разрез) текущей сборки;
   - пользовательские виды, созданные вручную.
4. Убедиться, что на плане не видны разрезы других сборок.
5. Открыть разрез одной из сборок.
6. Повторить проверку: видны только виды текущей сборки и пользовательские виды, чужие виды скрыты.
7. Создать вручную новый вид (не через плагин) и убедиться, что он остаётся видимым.
8. Повторно запустить плагин — старые виды должны замениться, фильтры не дублироваться.
