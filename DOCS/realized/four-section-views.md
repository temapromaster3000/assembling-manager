# План: 4 вида-разреза вместо 2

## Задача
Сейчас при создании видов создаётся два разреза с суффиксами `_Разрез_1` и `_Разрез_2`.
Требовалось сделать возможность создания любого из четырёх видов:
- Вид спереди
- Вид сзади
- Вид справа
- Вид слева

## Реализованные изменения

### Модель настроек (`src/AssemblingManager.Core/Models/ViewCreationOptions.cs`)
- Убран единый флаг `CreateSection`.
- Добавлены флаги:
  - `CreateFrontView`
  - `CreateBackView`
  - `CreateRightView`
  - `CreateLeftView`

### UI (`src/AssemblingManager.Revit/Views/MainWindow.xaml`)
- Вместо одной галочки «Разрез» добавлены 4 галочки:
  - Вид спереди
  - Вид сзади
  - Вид справа
  - Вид слева
- Галочки «План» и «3D вид» оставлены.

### Обработка UI (`src/AssemblingManager.Revit/Views/MainWindow.xaml.cs`)
- По умолчанию все 4 новые галочки включены.
- При нажатии «Сформировать» заполняются 4 новых флага.
- Проверка, что выбран хотя бы один вид, учитывает все 6 галочек.

### Сервис видов (`src/AssemblingManager.Revit/Services/ViewService.cs`)
- Добавлены суффиксы:
  - `_Вид спереди`
  - `_Вид сзади`
  - `_Вид справа`
  - `_Вид слева`
- Реализованы 4 метода создания разрезов:
  - `CreateFrontView`
  - `CreateBackView`
  - `CreateRightView`
  - `CreateLeftView`
- Удаление старых видов настраивается только по новым именам.
- Старые имена `_Разрез_1` и `_Разрез_2` не удаляются автоматически.

### Оркестратор (`src/AssemblingManager.Revit/Services/OrchestratorService.cs`)
- Вместо одного блока `if (options.CreateSection)` реализованы 4 независимых `if`, по одному на каждый вид.

## Направления видов
- **Вид спереди** — камера смотрит в сторону уменьшения Y (вдоль −Y).
- **Вид сзади** — камера смотрит в сторону увеличения Y (вдоль +Y).
- **Вид справа** — камера смотрит в сторону увеличения X (вдоль +X).
- **Вид слева** — камера смотрит в сторону уменьшения X (вдоль −X).

## Контрольный список ручной проверки в Revit

1. Открыть тестовую модель с несколькими `AssemblyInstance`.
2. Нажать кнопку «Сформировать виды» на вкладке Assembling Manager.
3. Убедиться, что в окне появились галочки:
   - План
   - Вид спереди
   - Вид сзади
   - Вид справа
   - Вид слева
   - 3D вид
4. Выбрать только «Вид спереди» и нажать «Сформировать».
   - Для каждой сборки должен появиться вид с именем `{ИмяСборки}_Вид спереди`.
5. Проверить, что ориентация вида соответствует описанному направлению.
6. Повторить шаги 4–5 для «Вид сзади», «Вид справа», «Вид слева».
7. Выбрать все виды и убедиться, что для каждой сборки создаётся 6 видов с правильными именами.
8. Повторно запустить команду с выбранными видами и проверить, что старые виды с такими же именами заменяются без ошибок.
9. Убедиться, что виды с именами `_Разрез_1` и `_Разрез_2` не удаляются автоматически.
10. Открыть несколько видов и проверить, что фильтр по сборке применён и на виде отображаются только элементы соответствующей сборки.
