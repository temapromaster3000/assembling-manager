# Реализовано: создание плановых видов сборок на уровне 0

## Статус

Реализовано.

## Проблема

При создании планового вида сборки плагин искал уровень, расположенный ниже bounding box сборки. Если такого уровня не находилось (например, в тестовом проекте нет уровней или сборка расположена ниже всех уровней), `ViewPlan.Create` получал `null`/`ElementId.InvalidElementId` и выбрасывал ошибку:

> The input argument "levelId" of function Autodesk::Revit::Proxy::DB::ViewPlanProxy::Create or one item in the collection is null.

## Решение

Плановый вид создаётся на уровне с отметкой 0:

- Если в проекте есть уровень с отметкой 0 — используется первый попавшийся.
- Если такого уровня нет — создаётся уровень с именем **AM_Отметка +0.000** на отметке 0.
- Созданный уровень не удаляется.
- **View Range** планового вида настраивается по bounding box сборки с запасом **±5000 мм**.
- **CropBox** планового вида по Z привязывается к bounding box сборки с запасом **500 мм**.

## Файлы, которые изменены

| Файл | Изменение |
|------|-----------|
| `src/AssemblingManager.Revit/Services/AssemblyService.cs` | Добавлен метод `GetOrCreateZeroLevelId`, удалён `GetNearestLevelId` |
| `src/AssemblingManager.Revit/Services/ViewService.cs` | Настроен `ViewRange` и `CropBox` по Z в `CreatePlanView` |
| `src/AssemblingManager.Revit/Services/OrchestratorService.cs` | Используется `GetOrCreateZeroLevelId` вместо `GetNearestLevelId` |

## Поведение

- Плановый вид создаётся всегда, когда в проекте есть хотя бы возможность создать уровень.
- В проекте без уровней появляется один уровень `AM_Отметка +0.000`.
- Повторный запуск не создаёт дубликат уровня.
- Элементы сборки видны на плане независимо от их реальной отметки.

## Контрольный список ручной проверки

- [ ] В проекте с уровнем 0 план создаётся на этом уровне.
- [ ] В проекте без уровня 0 создаётся уровень `AM_Отметка +0.000`.
- [ ] Повторный запуск не создаёт дубликат уровня.
- [ ] Плановый вид видит все элементы сборки.
- [ ] View Range корректно охватывает сборку с запасом ±5000 мм.
- [ ] CropBox по Z привязан к bounding box сборки.
- [ ] Если в проекте несколько уровней с отметкой 0 — берётся первый попавшийся.

## Примечания

В Revit API длина передаётся в футах. Для перевода миллиметров в футы используется коэффициент `304.8`:

```csharp
double feet = millimeters / 304.8;
```

Запас View Range: ±5000 мм ≈ 16.4 фута.
Запас CropBox: 500 мм ≈ 1.64 фута.
